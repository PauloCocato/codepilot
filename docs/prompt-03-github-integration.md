# Prompt 3: GitHub Integration (Issues, Repos, PRs)

> Cole este prompt no Claude Code após completar o Prompt 2.

---

## Prompt

```
Implemente a camada de integração com GitHub para o CodePilot.

## Contexto
Leia o CLAUDE.md para convenções do projeto.
O LLM adapter layer já está implementado em apps/agent/src/llm/.

## O que implementar:

### 1. apps/agent/src/github/issues.ts
Parser de issues do GitHub:
- `parseIssue(octokit, owner, repo, issueNumber)` → ParsedIssue
- Extrai do body: stepsToReproduce, expectedBehavior, actualBehavior
- Extrai file paths mencionados usando regex (ex: `src/index.ts`, caminhos com backticks)
- Extrai linguagem/framework mencionados
- Extrai code blocks do body (para contexto)
- Detecta tipo de issue: bug, feature, refactor, test, docs (baseado em labels + conteúdo)
- Valida com Zod: se campos obrigatórios estão presentes

### 2. apps/agent/src/github/repos.ts
Gerenciamento de repositórios:
- `cloneRepo(owner, repo, targetDir)` → { path, defaultBranch }
  - Usa simple-git com shallow clone (--depth 1) para speed
  - Clona em diretório temporário sob /tmp/codepilot/
- `createBranch(repoPath, branchName)` → void
  - Branch naming: `codepilot/issue-{number}-{slug}`
  - Slug: primeiras 5 palavras do título, kebab-case, max 50 chars
- `applyPatch(repoPath, patch)` → { success, filesChanged, error? }
  - Aplica unified diff ao repo
  - Retorna lista de arquivos modificados
- `commitAndPush(repoPath, message, branch)` → { sha }
- `cleanup(repoPath)` → void (deleta o clone)

### 3. apps/agent/src/github/prs.ts
Criação de Pull Requests:
- `createPR(octokit, params)` → { prUrl, prNumber }
  - params: { owner, repo, branch, title, body, issueNumber, labels? }
  - Title: "fix(codepilot): resolve #{issueNumber} — {issueTitleShort}"
  - Body template em markdown:
    ```
    ## Summary
    {explanation do agent}

    ## Changes
    {lista de arquivos modificados}

    ## Issue
    Resolves #{issueNumber}

    ## Agent Metadata
    - Attempts: {n}
    - Cost: ${cost}
    - Provider: {provider}
    - Time: {duration}

    ---
    *Generated by [CodePilot](link) — AI-powered issue resolution*
    ```
  - Adiciona label "codepilot" ao PR
- `commentOnIssue(octokit, owner, repo, issueNumber, body)` → void
  - Comenta na issue linkando o PR ou reportando falha

### 4. apps/agent/src/github/app.ts
Webhook handler para o GitHub App:
- Usa @octokit/webhooks para verificar signatures
- Escuta eventos:
  - `issues.labeled` — quando label "codepilot" é adicionada
  - `issues.opened` — se tem label "codepilot" desde a criação
  - `issue_comment.created` — se comentário contém "@codepilot" ou "/codepilot"
- Para cada evento, cria um job na fila BullMQ com: { owner, repo, issueNumber, trigger }
- Rate limiting: máximo 5 issues simultâneas por repositório

### 5. Testes
- issues.test.ts: parseia issues reais (use fixtures com bodies de exemplo)
  - Issue com bug report completo
  - Issue minimalista (só título)
  - Issue com code blocks e file mentions
- repos.test.ts: mock do simple-git, testa clone, branch, patch
- prs.test.ts: mock do Octokit, testa criação de PR com template correto
- app.test.ts: testa webhook handler com payloads de exemplo do GitHub
- Pelo menos 15 test cases no total

## Fixtures a criar (apps/agent/tests/fixtures/):
- github-issue-bug.json — payload de issue tipo bug com steps to reproduce
- github-issue-minimal.json — issue com só título e 1 linha de body
- github-issue-feature.json — feature request
- github-webhook-labeled.json — evento issues.labeled
- github-webhook-comment.json — evento issue_comment.created com "/codepilot"

## Regras:
- Nunca hardcode tokens — sempre via config/env vars
- Use Zod para validar payloads de webhook
- Logue cada operação de Git com Pino
- Trate erros do GitHub API (404, 403, rate limit) com mensagens claras
- Rode `npm run test` e `npm run typecheck` — ambos devem passar
- Commit: "feat: implement GitHub integration (issues, repos, PRs, webhooks)"
```

---

## Resultado esperado
Integração completa com GitHub: ler issues, clonar repos, criar branches, abrir PRs, e receber webhooks. Testado com fixtures.
